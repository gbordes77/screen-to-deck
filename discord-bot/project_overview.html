📋Résumé du projet

Screen to Deck convertit automatiquement des captures d'écran de decks Magic: The Gathering (MTG) en listes prêtes à l'import sur MTGA, MTGO, Moxfield, etc. Il utilise l'OCR, la validation Scryfall, et exporte dans tous les formats populaires, avec monitoring et tests automatisés.

  • Stack : Python, Discord.py, Tesseract/Azure OCR, Scryfall API, Pytest, Sphinx, Docker, GitHub Actions, Grafana
  • Exports : MTGA, MTGO, Moxfield (avec codes d'édition), CSV, JSON
  • Monitoring : Logs, CSV, Dashboard Grafana

🔄Schéma des processus

Workflow principal :
🖼️[Image Deck] → 🔎[OCR & Parsing] → 🗂️[Validation Scryfall] → 🛠️[Correction] → 📤[Export Multi-format]
                                                    ↓
                                            📊[Monitoring & Logs]
  • 🔎OCR & Parsing : Extraction des noms/quantités via Tesseract ou Azure, préprocessing OpenCV
  • 🗂️Validation Scryfall : Recherche exacte/fuzzy, suggestions, correction automatique
  • 📤Export : Génération de fichiers MTGA, MTGO, Moxfield, CSV, JSON
  • 📊Monitoring : Temps de réponse, cache hits/misses, export CSV, dashboard Grafana

🏗️Architecture & Organisation

  • discord-bot/ : Backend (bot, OCR, Scryfall, export, monitoring)
  • tests/ : Tests unitaires Pytest (parser, Scryfall, export)
  • docs/ : Documentation Sphinx, dashboard Grafana, screenshots
  • .github/workflows/ : CI/CD GitHub Actions (lint, tests, doc auto)

Chaque composant est modulaire et facilement remplaçable (ex : provider OCR, format d'export, etc.).
