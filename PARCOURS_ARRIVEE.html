Tout lancer (one‑button)

Bootstrap → Redis (si dispo) → API+Front → Health → URLs

./scripts/one-button.sh

Pas de config manuelle: le script détecte l’IP LAN, configure CORS, et lance front+API en arrière‑plan.

Vérifier
  • UI: http://<IP_LAN>:5173
  • API: /health, /api/health
  • Docs API: /api/docs
  • Metrics: /metrics
Valider E2E

Ouvre TEST_PLAN.html et suis le pas‑à‑pas (upload OCR → validation Scryfall (batch) → export).

Contribuer
  • Petites PR testées; mets à jour doc/changelog si besoin
  • Paramètres (optionnels) dans server/.env: MAX_JOBS_PER_HOUR, MAX_DAILY_COST_EUR, OCR_CONCURRENCY
Avant livraison / fin de mission
  • Vérifie CHANGELOG.md, teste ROLLBACK.md
  • Checklist passation: MISSION_CLOSEOUT.md

Besoin d’aide rapide? Démarre par ./scripts/one-button.sh puis ouvre TEST_PLAN.html.
